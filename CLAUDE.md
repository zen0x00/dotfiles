## What This Is

A dotfiles repository for automating a Hyprland Wayland desktop environment on Arch Linux. It installs packages, symlinks configs via GNU Stow, and applies themes through a template substitution system called **zen0x**.

## Installation Commands

```bash
./bootstrap.sh    # Interactive installer тАФ handles packages, stow, theming, monitor config
./install.sh      # Bare-metal Arch Linux OS install (disk partitioning, btrfs, bootloader) тАФ NOT for dotfiles
```

### Modular Re-runs

```bash
modules/stow.sh              # Re-symlink all dotfiles to ~/.config/
modules/default-theme.sh     # Re-apply the default (Abyssal) theme
modules/monitors.sh          # Regenerate hypr/configs/monitors.conf
zen0x-theme-generate NAME    # Regenerate theme configs from templates
zen0x-apply-generated-theme  # Apply generated configs to running apps
```

## Architecture

### Installation Flow (`bootstrap.sh`)

1. Sources `utils/logging.sh` and `utils/checks.sh`
2. Prompts: machine type (Desktop/Laptop) and install mode (Minimal/Full)
3. Runs modules sequentially: packages тЖТ git config тЖТ stow тЖТ zsh тЖТ monitors тЖТ default theme тЖТ reboot

### Module System (`modules/`)

13 independent bash scripts, each sourcing utils and performing one setup phase. Key modules:
- `packages-core.sh` тАФ 111 essential packages via `yay`
- `packages-extra.sh` тАФ 60 optional packages (dev tools, gaming, media) for Full mode only
- `stow.sh` тАФ symlinks 14 config directories into `~/.config/` using GNU Stow
- `default-theme.sh` тАФ links the Abyssal theme, runs zen0x generators, applies GTK/VSCode/Neovim themes
- `bin.sh` тАФ symlinks scripts from `bin/` to `/usr/bin`

### Theme System (`zen0x`)

Template-driven color scheme application:
- **Source**: `themes/colorschemes/<NAME>/colors.toml` тАФ TOML color definitions
- **Templates**: `zen0x/templates/*.tpl` тАФ config templates with `{{COLOR_VAR}}` placeholders
- **Output**: `~/.config/zen0x/generated/` тАФ generated configs (gitignored, regenerated at runtime)
- **Flow**: `zen0x-theme-generate` does sed substitution тЖТ `zen0x-apply-generated-theme` reloads apps

13 colorschemes available: Abyssal (default), Dracula, Catppuccin, Tokyo-Night, Gruvbox, and others.

### Hyprland Config (`hypr/`)

Modular structure тАФ `hyprland.conf` sources 11 sub-configs from `hypr/configs/`:
`animations`, `apps`, `envs`, `execs`, `general`, `gestures`, `inputs`, `keybinds`, `monitors`, `rules`, `plugins`

Machine-specific files are gitignored: `hypr/configs/monitors.conf`, `hypr/configs/colors.conf`

### Utility Scripts (`bin/`)

18 executables prefixed `zen0x-*` plus `perf-mode` and `theme-doctor`:
- `zen0x-theme-*` scripts handle per-app theme application (gtk, wallpaper, nvim, vscode, reload)
- `zen0x-menu` тАФ interactive theme/app launcher
- `perf-mode` тАФ toggles Hyprland performance mode (disables animations/blur/shadows/VRR)
- `theme-doctor` тАФ validates theme integrity

### Utility Libraries (`utils/`)

- `logging.sh` тАФ color-coded `log_info`, `log_warn`, `log_error`, `log_success` functions
- `checks.sh` тАФ prerequisite validation (yay present, Arch-based OS, etc.)

All modules and scripts source these at the top.

## Gitignored / Machine-Specific Files

These are generated locally and not tracked:
- `hypr/configs/monitors.conf` тАФ generated by `modules/monitors.sh`
- `hypr/configs/colors.conf` тАФ generated by zen0x theming
- `zen0x/generated/` тАФ all generated application configs
- `themes/colorschemes/current` тАФ symlink to active theme
- `nvim/plugins/colorscheme.lua` тАФ generated by theme system
