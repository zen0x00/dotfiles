## What This Is

A dotfiles repository for automating a Hyprland Wayland desktop environment on Arch Linux. It installs packages, symlinks configs via GNU Stow, and uses **matugen** for colorscheme generation.

## Installation Commands

```bash
./bootstrap.sh    # Interactive installer — handles packages, stow, theming, monitor config
./install.sh      # Bare-metal Arch Linux OS install (disk partitioning, btrfs, bootloader) — NOT for dotfiles
```

### Modular Re-runs

```bash
modules/stow.sh              # Re-symlink all dotfiles to ~/.config/
modules/monitors.sh          # Regenerate hypr/configs/monitors.conf
zen0x-mode minimal           # Switch to minimal mode
zen0x-mode eye-candy         # Switch to eye-candy mode
```

## Architecture

### Installation Flow (`bootstrap.sh`)

1. Sources `utils/logging.sh` and `utils/checks.sh`
2. Prompts: machine type (Desktop/Laptop) and install mode (Minimal/Full)
3. Runs modules sequentially: packages → git config → stow → zsh → monitors → reboot

### Module System (`modules/`)

Independent bash scripts, each sourcing utils and performing one setup phase. Key modules:
- `packages-core.sh` — essential packages via `yay`
- `packages-extra.sh` — optional packages (dev tools, gaming, media) for Full mode only
- `stow.sh` — symlinks config directories into `~/.config/` using GNU Stow
- `bin.sh` — symlinks scripts from `bin/` to `/usr/bin`

### Mode System (`modes/`)

Two visual modes that control layout/effects independently of colorschemes:
- **minimal** — flat, no effects, monochrome, solid backgrounds
- **eye-candy** — blur, shadows, animations, rounded corners, gradients

Each mode contains its own `hypr/`, `waybar/`, and `walker/` configs. Switched via `zen0x-mode <name>` or the Mode option in `zen0x-menu`.

### Colorscheme System

Uses **matugen** for colorscheme generation. Colors are independent of modes.

### Hyprland Config (`hypr/`)

Modular structure — `hyprland.conf` sources 11 sub-configs from `hypr/configs/`:
`animations`, `apps`, `envs`, `execs`, `general`, `gestures`, `inputs`, `keybinds`, `monitors`, `rules`, `plugins`

Machine-specific files are gitignored: `hypr/configs/monitors.conf`, `hypr/configs/colors.conf`

### Utility Scripts (`bin/`)

- `zen0x-menu` — interactive launcher (Wallpaper, Font, Mode)
- `zen0x-mode` — switches between minimal/eye-candy modes
- `perf-mode` — toggles Hyprland performance mode (disables animations/blur/shadows/VRR)

### Utility Libraries (`utils/`)

- `logging.sh` — color-coded `log_info`, `log_warn`, `log_error`, `log_success` functions
- `checks.sh` — prerequisite validation (yay present, Arch-based OS, etc.)

All modules and scripts source these at the top.

## Gitignored / Machine-Specific Files

These are generated locally and not tracked:
- `hypr/configs/monitors.conf` — generated by `modules/monitors.sh`
- `hypr/configs/colors.conf` — generated by colorscheme system
- `nvim/lua/plugins/colorscheme.lua` — generated by theme system
- `walker/themes/current/colors.css` — generated colors
- `swaync/colors.css` — generated colors
