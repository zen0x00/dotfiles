# nvim Theme Sync Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wire nvim into the zen0x theme engine so `zen0x-apply-theme <slug>` generates a colors Lua module and reloads all running nvim instances.

**Architecture:** A Jinja-style template renders `~/.config/nvim/lua/zen0x/colors.lua` (a plain Lua color table) from `colors.toml`. A static `nvim/colors/zen0x.lua` colorscheme file reads that table and applies all highlight groups. `zen0x-theme-reload` iterates nvim server sockets and sends `:colorscheme zen0x`.

**Tech Stack:** Python (existing template engine), Bash (existing reload script), Lua (nvim colorscheme)

### Task 6: Add nvim socket reload to zen0x-theme-reload

**Files:**
- Modify: `bin/zen0x-theme-reload`

- [ ] **Step 1: Open `bin/zen0x-theme-reload` and find the end of the reload steps**

Locate the line with `command -v pkill >/dev/null 2>&1 && pkill -HUP hyprlock 2>/dev/null || true`

- [ ] **Step 2: Add the nvim socket loop after the hyprlock line and before the notify-send block**

```bash
# reload running nvim instances
for _sock in /run/user/$(id -u)/nvim.*.0; do
    [[ -S "$_sock" ]] && nvim --server "$_sock" --remote-send ':colorscheme zen0x<CR>' 2>/dev/null || true
done
unset _sock
```
